1. Пользователь публикует посты в instagram / youtube 

2. Пользователь добавляет аккаунты социальных сетей в мое приложение, наделяя их правами для автоматической публикации

3. Пользователь может отправить со своего специального аккаунта сообщение на мой аккаунт с контентом - это будет триггером для создания нового контента на основе отправленного. Причем аккаунт, с которого отправят - будет определять сценарии, по которым будет готовиться контент

4. Пользователь может помотреть запланированные публикации.

5. Пользователь может отменить публикации.

6. Пользователь может выписать себе API KEY, чтобы использовать его для интеграции с другими сервисами.

7. Пользователь может создать организацию или несколько, добавить в них социальные аккаунты, и работать с организацией как единым аккаунтом.

8. [Contario] - пользователь хочет выбрать сценарий и создать видео на основе этого сценария. 

9. [Contario] - пользователь хочет опубликовать сделанное в проекте видео в социальной сети (Например инстаграмм)

10. Пользователь хочет авторизоваться через единый аккаунт (Keycloak) и иметь доступ ко всем сервисам (Publish, Content Factory, Contario) без повторного входа (SSO).

12. [Spam] - пользователь хочет создать спам-кампанию, указав целевые аккаунты, сценарий создания контента и расписание публикаций.

15. [Content Factory] - пользователь хочет создать пайплайн обработки видео (обрезка, наложение музыки, субтитры) и сохранить его как переиспользуемый сценарий.

17. Пользователь хочет назначать роли участникам организации (админ, редактор, зритель), чтобы разграничить доступ к аккаунтам и публикациям.

18. [Contario] - пользователь хочет кроссплатформенно публиковать одно видео одновременно в Instagram, TikTok и YouTube.
